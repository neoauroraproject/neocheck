# Contributing to NeoCheck

We welcome contributions of all kinds to NeoCheck! This guide helps you get started with contributing.

## How to Contribute

### 1. Reporting Bugs
- Search open issues to verify if the bug has already been reported.
- If not, create a new issue using the **Bug Report** template, providing clear reproduction steps.

### 2. Suggesting Features
- Open an issue describing the feature, why it is needed, and any implementation designs.

### 3. Submitting Code Changes (Pull Requests)
- Fork the repository and create your branch from `main`.
- Write unit tests for new logic.
- Ensure your code conforms to the Go formatting (`go fmt`) and Next.js linting guidelines.
- Submit a PR detailing the changes.
